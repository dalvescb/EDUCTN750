Part 1: How to access remote servers via ssh

Unix and Unix-Like (AKA Un*x) Systems

  • Unix is a family of Operating Systems that all conform to the Single UNIX Specification
  • The terminology Un*x or Unix-Like is used to refer to THE MANY systems that are very similar to Unix but don't fully conform to the standard
  • Examples of Unix-Like Systems include MacOS and Linux Variant Systems

Evolution of Un*x

Remote Login With SSH

  • From a terminal, one can ssh into a hypothetical domain http://somedomain.com with user account username like so
ssh username@somedomain.com
# enter password prompt
# do some work
exit

Investigate OS Info

  • Find out info about an OS with the following command
uname -a
  • Which will list the following info (in order)
    • kernel name
    • network hostname
    • kernel release, kernel version
    • machine, processor, hardware-platform
    • operating system

CHALLENGE

SSH into a server, investigate it's OS information and then exit

Part 2: How to Navigate/Manipulate a Un*x Filesystem

Navigating File Paths

  • File Paths specify file locations through directories seperated by a backslash
/path/to/file.txt
  • Special File Paths (certain characters are reserved for special paths)
cd /   # Root directory
cd ~   # Home directory
cd .   # Current directory
cd ..  # Previous directory

Basic File Browsing Commands

These commands will help you get around 

cd /path/to
  # change current directory

ls
  # List current directory contents

pwd
  # Show parent directory (where am I?)

Absolute vs Relative File Paths

  • Absolute file paths begin with a backslash
cd ~
cd /some/path
pwd 
  # /some/path
  • Relative file paths DO NOT begin with a backslash
cd ~
cd some/path
pwd
  # ~/some/path

File/Directory Manipulation

Basic file manipulation operations you might want to perform on a file / directory

  • Creation
  • Deletion
  • Copy
  • Move

File/Directory Creation

The following commands create a file / directory 

touch somefile.txt
  # create an empty file

mkdir somedir
  # create a new directory

Note: you can also create a new file using an code editor

File/Directory Deletion

The following commands delete a file / directory 

rm somefile.txt
  # deletes a file

rm -r somedir
  # delete a new directory

Warning: there's no coming back from rm

File/Directory Copy

The following commands copy a file / directory 

cp somefile1.txt somefile2.txt
  # copy a file

cp -r somdir1 somdir2
  # copy a directory

Note: the -r flag stands for recursive

File/Directory Move

The following commands move a file / directory 

mv somefile.txt ~/somefile.txt
  # moves a file to your HOME

mv dir ~/dir
  # works the same as file

Note: you can also use the mv command to rename a file

CHALLENGE

Try and perform the following steps

  • Create a directory called MyDir
  • Create another directory called MyDir2
  • Move MyDir2 into MyDir
  • Create a file called MyFile
  • Create a copy of MyFile in MyDir2

Part 3: How to create and execute a simple bash script

Command-line Based Text Editors

There's a decent selection of command-line based code editors

  • Emacs https://www.emacswiki.org
  • Vi/Vim https://www.vim.org
  • Nano https://www.nano-editor.org

Note: most desktop editors have some method of remote editing, so using terminal-based editors isn't always necessary

Bash Scripts

  • Create a file test.sh and open it in your preferred editor

  • Fill in the following code 

    #!/bin/bash

echo "Hello World!"

  • To execute the file, make it an executable with chmod 

    chmod +x test.sh

  • Then execute it from your shell by preceding the file with ./ 

    ./test.sh

She-Bangs!

  • You may have noticed the line beginning with #! at the top of our hello world program before
  • These lines are known as She-Bangs and are used to specify which interpreter to run a script with when using the ./ syntax

  • You can use this to run python files as scripts as well 

    #!/usr/bin/python

print("Hello World")
